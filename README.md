# 体調管理アプリ

自分の体調・食事・行動を記録して、不調の原因を統計的に分析するスマートフォンアプリです。

> **このアプリは [Claude Code](https://claude.ai/code) を使って開発しました。**

---

## 機能

### 記録
- **体調記録** — 1日に複数回、時間帯ごとにスコア・症状・メモを記録
- **行動・食事記録** — カスタムカテゴリで自由に記録（食事、アルコール、睡眠、運動など）
- **過去日入力対応** — 忘れた日の記録も遡って入力できる
- **時間帯選択** — 朝・午前・昼・午後・夕方・夜・深夜の7区分でざっくり入力

### カスタムカテゴリ
- 自由記述・選択式（ラジオ/プルダウン）を自分で設定
- デフォルトカテゴリ（食事・アルコール・睡眠・運動・カフェイン・ストレス）付き
- 選択肢は1行1つで自由に追加

### 分析
- **ラグ相関分析** — 最大14日前の行動・食事と体調スコアの相関を自動計算
- **累積効果検出** — 「3日連続アルコール → 不調」のようなパターンも検出
- 5件以上の記録が溜まると分析が有効になる

詳しくは [分析手法について](#分析手法について) を参照。

### セキュリティ
- **生体認証ロック** — アプリ起動時に Face ID / 指紋認証を要求。認証に失敗するとデータにアクセスできない
- **暗号化ストレージ** — 全データを [expo-secure-store](https://docs.expo.dev/versions/latest/sdk/securestore/) で暗号化保存。iOS は AES-256（Secure Enclave）、Android は AES-256-GCM（Keystore）を使用
- **データはローカルのみ** — 健康データはクラウドに送信されない。アカウント登録不要

## セキュリティ

体調・食事などの個人データを安全に扱うため、3層のセキュリティを実装しています。

| 機能 | 実装 | 効果 |
|------|------|------|
| **生体認証ロック** | expo-local-authentication（Face ID / 指紋認証） | アプリ起動時に本人確認。他人がスマホを操作してもデータを見られない |
| **暗号化ストレージ** | expo-secure-store → iOS: AES-256（Secure Enclave）/ Android: AES-256-GCM（Keystore） | 全記録データをOSレベルで暗号化。ファイルを直接読んでもデータを解読できない |
| **データはローカルのみ** | アプリコード内にネットワーク通信なし | 健康データはクラウドに送信されない。アカウント登録不要 |

> **補足**: Expo frameworkのOTAアップデート確認など、フレームワークレベルの通信は発生する場合があります。アプリが収集した健康データがネットワークに送信されることはありません。

## セットアップ

### 必要なもの
- Node.js 18以上
- スマートフォン（[Expo Go](https://expo.dev/go) アプリをインストール）

### 手順

```bash
# リポジトリをクローン
git clone https://github.com/halso0218/health-tracker.git
cd health-tracker

# 依存関係をインストール
npm install

# 開発サーバーを起動
npx expo start
```

表示されたQRコードをスマートフォンのカメラで読み込むと起動します。

## 分析手法について

### ピアソン積率相関係数 × 時間差相関分析（Lag Correlation Analysis）

通常の相関分析は「同じ日のデータ」同士を比較しますが、体調への影響は翌日・数日後に現れることがあります。このアプリでは**時間をずらして（ラグをかけて）** 相関を計算することで、「何日前の行動が今日の体調に影響しているか」を特定します。

```
ラグ0:  アルコール(1/15) × 体調(1/15)  → 当日の影響
ラグ1:  アルコール(1/15) × 体調(1/16)  → 翌日の影響
ラグ2:  アルコール(1/15) × 体調(1/17)  → 2日後の影響
...
ラグ14: アルコール(1/15) × 体調(1/29)  → 14日後の影響
```

全ラグで相関係数（-1〜+1）を計算し、最も強い負の相関（体調低下との関係）を持つ因子と日数を「有力な原因」として表示します。

### 累積効果の検出

「3日連続で睡眠不足 → 今日不調」のような蓄積型パターンは、単純なラグでは拾えません。「前N日間ずっとその行動があったか（0/1）」と「今日の体調スコア」の相関を別途計算することで、こうした累積効果も検出します。

### 相関係数の解釈

| 相関係数 | 表示 | 意味 |
|---------|------|------|
| -0.6以下 | 🔴 強い相関 | その行動と不調に強い関係がある |
| -0.3〜-0.6 | 🟡 中程度の相関 | 傾向として現れている |
| -0.3以上 | 表示しない | 関係が弱いため除外 |

### 注意事項

- **相関は因果ではありません。** 「アルコールを飲んだ翌日に不調」でも、「飲み会の翌日は睡眠が短い」など別の原因が隠れている場合があります。
- データが少ないと相関係数が不安定です（最低5件、理想は30件以上）。
- 複数の原因が重なる場合（交絡因子）は分離できません。
